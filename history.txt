
-[ Версия 1.3 ]---------------------------------------------------------------

Все изменения (C) Rond, 2019.


   1. Исправления в коде игры.

 1.1. При старте новой игры теперь очищается флаг начала войны из-за отказа
      жениться.
 1.2. Шаман теперь лечит при положительном ответе на соответствующий вопрос.
 1.3. Убрана очистка экрана при инициализации новой игры, поскольку экран и
      так очищается в начале каждого хода.
 1.4. Попытка избавиться от переполнения буфера при использовании scanf.


   2. Изменения.

 2.1. Убраны вопросы насчёт новогоднего бала, т.к. ответы всё равно ни на что
      не влияют.[1]
 2.2. Убраны просьбы королевы, т.к. ответы всё равно ни на что не влияют.[1]
 2.3. При выводе сообщения о смерти королевы больше не спрашивается впускать
      ли гонца.[1]
 2.4. На еду выделяется 70% от необходимого. Этого достаточно чтобы избежать
      революции.[1]
 2.5. Убрано предупреждение о революции если норма зерна < 100% но >= 70%.
 2.6. Цены генерятся в начале каждого хода, а не в начале торговли на бирже.
 2.7. В начале каждого хода автоматически пишутся биржевые цены.
 2.8. Добавлено автосохранение игры. РУЧНОГО СОХРАНЕНИЯ НЕТ. Игра
      автоматически сохраняется перед началом каждого хода, кроме первого. При
      старте новой игры, автоматически создается новый подкаталог для
      сохранения игры в каталоге SAVE, При продолжении старой игры,
      существующие файлы могут автоматически перезаписываться.
 2.9. Слегка изменена заставка и обновлена информация о версии игры.
2.10. В комплекте поставляется также исполняемый файл Win32, для его
      компиляции используется Tiny C Compiler.
2.11. Используется новый парсер командной строки.
2.12. Добавлен параметр командной строки "vanilla" (без кавычек), он
      используется для включения режима игры как в оригинале (будут вопросы
      насчёт новогоднего бала, просьбы королевы, будет спрашиваться впускать
      ли гонца от королевы и при автоматическом распределении зерна норма 100%
      вместо 70%).
2.13. В главном меню выводится информация о том, включен или нет режим
      оригинальной игры. Кроме того, можно переключить этот режим прямо из
      главного меню.
2.14. Добавлен параметр командной строки для загрузки игры. Для этого
      достаточно указать имя файла сохранения, например "SAVE\2.SAV" (без
      кавычек). У файла сохранённоый игры обязано быть расширение.
2.15. Добавлен параметр командной строки "noautosave" (без кавычек), он
      используется для выключения автоматического сохранения игры.
2.16. В главном меню выводится информация о том, включен или нет режим
      автосохранения игры. Кроме того, можно переключить этот режим прямо из
      главного меню.
2.17. Числа в таблице биржевых цен форматируются по правому краю.
2.18. В таблице биржевых цен сообщается максимальное количество товаров
      каждого вида, которые можно купить.
2.19. Теперь используется одна и та же функция для вывода информациия в начале
      каждого хода и для вывода информации при торговле на бирже.
2.20. Вывод заставки можно прервать, нажав любую клавишу.
2.21. Игра автоматически создает каталог "SAVE" (без кавычек), если его не
      было.
2.22. Убран флаг fl_block, т.к. он ни на что не влияет.
2.23. Добавлен ключ командной строки "loadlast" (без кавычек), он используется
      для загрузки последнего файла автосохранения.
2.24. Добавлен пункт меню для загрузки последнего файла автосохранения.
2.25. Пишем просто "Мобилизовано .. человек." вместо "Мобилизовано .. человек.
      В народе растет недовольство!".
2.26. Одна и та же функция используется, если:
      - выиграна война;
      - проиграна война
      - получено наследство.
2.27. При прибытии каравана очищаются флаги:
      - fl_kar (это позволяет отправлять караван несколько раз за игру);
      - for_kar (это позволяет немного уменьшить развер файла сохранения).
2.28. Больше не требуется нажимать клавишу перед началом нового хода.
2.29. Не требуется нажимать клавишу после:
      - собщения о международном кризисе;
      - митрополита;
      - сообщения о поимке визиря;
      - сообщения о наследстве;
      - сообщения о родившемся сыне;
      - женитьбы;
      - сообщения о смерти жены.
2.30. Используется специальная функция для считывания нажатой клавиши. Это
      нужно для нормальной обработки нажатия курсорных клавиш как под DOS, так
      и под Windows.
2.31. Переписана функция torgovla(), исходный текст стал гораздо короче.
2.32. Если отправлен караван, то пишется информация о том, через сколько ходов
      он прибудет.
2.33. Пишется информация о том, сколько требуется зерна.
2.34. Добавлен параметр командной строки "new". Он используется для старта
      новой игры.
2.35. Исправлены орфографические ошибки.
2.36. Склоняются слова с разных фразах.
